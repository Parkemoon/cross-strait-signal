# Frontend

React 19 + Recharts + Tailwind CSS 4. State management lives in `App.js`. All API calls use relative URLs (`API_BASE = ""`); dev server proxies `localhost:8000` via `"proxy"` in `package.json`.

## Central API client (`api.js`)

Every fetch in the frontend goes through a named function in `api.js`, not inline `fetch`. When adding a new API endpoint, add the corresponding function to `api.js` first; components import from it directly.

`fetchStats` only forwards keys in `SCOPING_KEYS` to the stats endpoint (`sentiment` and `search` are intentionally article-list-only). When adding a new scoping filter, add it to both `SCOPING_KEYS` here AND `_build_filter_clause` in `stats.py`.

## Read-only build (`readOnly.js`)

`READ_ONLY = process.env.REACT_APP_READ_ONLY === 'true'`. Import in any component with write controls to hide them in the public build. Build with `npm run build:public` (sets `BUILD_PATH=build-public`). Nginx also blocks POST/PATCH on the public server at the edge.

## Layout and view state (`App.js`)

- `view` state (`"feed"` | `"review"` | `"economy"` | `"trade"` | `"people"` | `"military"` | `"polls"` | `"diplomacy"`) controls the center column on desktop. `"review"` → `<ReviewQueue />`; `"economy"` → `<EconomyTab />`; `"trade"` → `<TradeAccessTab />`; `"people"` → `<PeopleTab />`; `"military"` → `<MilitaryTab />`; `"polls"` → `<PollsTab />`; `"diplomacy"` → `<DiplomacyTab />`.
- The 3-column grid collapses to 2 columns (hides the Social Pulse right aside) when `view` is `"economy"`, `"trade"`, `"people"`, `"military"`, `"polls"`, or `"diplomacy"`, so wide tables/charts/maps get full width.
- Below 768px (`isMobile`, from `hooks/useWindowWidth.js`), everything collapses to a single column with a sticky top tab bar (`mobileTab` state). Each tab shows/hides its panel via `display: none`. Check `isMobile` when adding any fixed-width or multi-column structure.

## Sync points — keep these in lockstep when adding things

| Adding | Update |
|---|---|
| A scoping filter | `SCOPING_KEYS` (api.js) + `_build_filter_clause` (stats.py) + `hasScopingFilter`/`buildScopeLabel` (StatsSidebar) |
| A new source place | API filter block in `articles.py`; never hardcode beyond PRC / TW / hk / intl in `FilterBar` |
| A multi-feed publication | `PUBLICATION_NAMES` (StatsSidebar source grouping) + `SOURCE_FILTER` (click-to-filter prefix map) + `SOURCE_ABBREV` (SourceBadge) |
| A TW-vs-PRC pair on Economy tab | `MACRO_PAIRS` in `EconomyTab.jsx` (set `dualAxis: true` when scales differ ≥5×) |
| A verification reporter pair | `VERIFICATION_PAIRS` (economy.py) + `VERIFICATION_KINDS` (EconomyTab.jsx) for subsection styling |
| A new industry that misses bar colour | Widen the `INDUSTRY_SECTOR` token list in EconomyTab.jsx or accept grey "Other" |
| A new trade-access direction or status | `DIRECTIONS` / `STATUS_FILTERS` / `STATUS_PILLS` in TradeAccessTab.jsx |
| A new economic indicator | `SERIES_META` in `api/routes/economy.py` with a `category` field — picker auto-discovers via `data.series` (except `category='macro'`, which is filtered out and goes through MacroSection instead) |
| A new metric for `cross_strait_population` | Update the seed script or scraper to produce the metric; `PeopleTab.jsx` pulls a small fixed set of metrics (latest residence permit, census, spouse stock, 台胞证 holders) so add a KPI card and/or chart for new metrics that should be surfaced |

## Component-specific notes

- **`StatsSidebar.jsx`**: dashboard gauges sorted PRC → TW → HK/Macao → International. Taiwan camp gauges (`green` / `green_leaning` / `blue`) hidden below n=5 articles to avoid noise. When a scoping filter is active, a teal chip appears above "Strait Watch" with a dismissable `×`; each gauge shows a grey ghost dot at the global baseline position (only when scoped score differs by >0.01). `TopicBreakdownChart` hides when `filters.topic` is set. All sidebar elements are clickable to set filters: gauges → `onPlaceClick(placeKey|null)`, camp gauges → `onBiasClick(bias)`, source rows → `onSourceClick(dbPrefix)` using `SOURCE_FILTER` map, entity rows → `onEntityClick(entityNameEn)`. Renders `<StatSpotlight onOpen={onOpenTab} />` between the topic breakdown and Sources sections; `onOpenTab(view)` is the generic tab-switch callback from `App.js` (replaced the old economy-only `onOpenEconomy`).
- **`StatSpotlight.jsx`**: rotating single-slot sidebar widget — one card cycles a headline figure from each top-level section (Economy trade balance → PLA 30-day ADIZ avg → NCCU Taiwanese-identity % → PRC ECFA suspensions → monthly cross-strait visitor flow). Each slide-builder deliberately picks the *freshest* meaningful figure its section exposes — e.g. People uses the monthly visitor flow, not the spouse/permit stocks which lag years (the NIA 13503 spouse resource is frozen at 2020). Replaced the old static `EconomyMini`; economy is slide 0 so first paint is unchanged. Fetches all five section summaries in parallel on mount (`Promise.allSettled`), each via a per-section *slide-builder* that returns `null` when its data isn't ready, so a slow/empty section silently drops out of the rotation. Auto-advances every `AUTO_MS` (7s) with pause-on-hover/focus and pause-when-offscreen (IntersectionObserver — also covers the mobile `display:none` collapse); honours `prefers-reduced-motion` (no auto-advance). Dot nav steps manually + re-arms the timer via `cycleNonce`. Whole card + "More →" call `onOpen(view)`. The poll slide matches the "Taiwanese" option by **label** (`label_en === "Taiwanese"`), not position. Self-contained formatters (`formatUsdB`/`formatCount`/`DeltaChip`) — does NOT import EconomyTab's (kept decoupled). To retune which stat a section surfaces, edit that section's slide-builder.
- **`ArticleCard.jsx`**: inline sentiment/topic override + analyst notes. `onSignalOff` prop for FlashTraffic removal; `onApprove` callback for pending count updates. Unapproved articles (`analyst_approved=0`) show an amber left border and Approve/Dismiss banner (admin only). `FieldEditor` handles inline editing of the three `*_override` columns — pencil icon reveals textarea; overridden fields render in amber. `sentiment_reasoning` renders as a small italic grey line below the sentiment badge (admin only, hidden when empty).
- **`ReviewQueue.js`**: human review UI with translation editing fields (headline, summary, key quote) always visible — changed fields saved via `updateArticleTranslation` before resolving. Confirm/override auto-approves the article.
- **`SignalCharts.jsx`**: sentiment trend (Y-axis clamped to `[-1, 1]`, single YAxis) + topic breakdown.
- **`EconomyTab.jsx`**: KPI strip (4 cards), main trade chart with 1Y/3Y/5Y/All range toggle, indicator picker (macro filtered out — has its own section), `MacroSection` for TW-vs-PRC side-by-side line charts driven by `MACRO_PAIRS`, `VerificationSection` rendering one subsection per `kind` returned by `/api/economy/verification` (styling per kind in `VERIFICATION_KINDS`), `InvestmentSection` (default direction `tw_to_prc` — the dominant flow). Verification charts always show last 60 months regardless of main-chart range (PRC data lags ~6 months). Investment bars colour-coded by `INDUSTRY_SECTOR` via `classifySector(zh)`; `formatInvestmentAmount` adapts K/M/B (outbound is ~50× inbound). (The sidebar trade-balance widget moved out into `StatSpotlight.jsx` — EconomyTab no longer exports `EconomyMini`.)
- **`PeopleTab.jsx`**: own top-level tab as of 2026-05-23 (was a section inside EconomyTab). KPI strip (4 cards), bidirectional 2-column grid (collapses to 1 column under ~680px via `grid-template-columns: repeat(auto-fit, minmax(340px, 1fr))`): LEFT = PRC-in-TW (grouped bar chart of annual residence + settlement permits); RIGHT = TW-in-PRC (milestone cards + vertical policy timeline). Flow strip = paired monthly visitor flows (TW→PRC dark vs PRC→TW teal) — shows ALL available periods rather than slicing to last 60, because MAC only publishes TW outbound from 2024-03 and slicing made the line look like it cliffed. Closing methodology blurb explains why the two sides are asymmetric (1992 籍貫 cutoff + no 出生地 publish, see [[tw-birthplace-data-unavailable]]). Reads `/api/economy/people-records`. Local `formatLargeNumber` deliberately keeps 1 decimal at the 10M+ band (so 11.6M renders as "11.6M" not "12M") — the policy-timeline "12M+ targeted" wording is a different concept from cumulative holders and the two figures should be visibly different.
- **`TradeAccessTab.jsx`**: Headline strip framing the asymmetry, suspension wave timeline (per `summary.suspension_waves`), filter bar (direction toggle + status pills + debounced search), paginated table sorted by status severity. `STATUS_PILLS` colour scheme — `banned`=red, `ecfa_suspended`=amber, `conditional`=blue, `ecfa_active`=green — the contrast between directions IS the analytical point; don't soften it.
- **`MilitaryTab.jsx`**: Top-level tab, purple-tinted ("hostile" colour). Two main sections: **PLA Incursions** (KPI strip, daily bars, ADIZ sector heatmap, custom Taiwan Strait SVG map built via `scripts/build_taiwan_strait_map.py` — re-run that only to retune resolution, bbox, viewBox, or island roster) and **Exercise Tracker** (Leaflet map + scrollable list + analyst review queue + edit modal). MND-only fields (vessels, coast-guard, zone breakdown) render only for post-2022 periods because PLATracker backfill never published them — see [[mnd-incursion-parsing]] and the database rule on `pla_incursions`. PERFORMER_COLOUR / PERFORMER_LABEL maps live in `ExerciseMap.jsx` and are re-imported here for the filter pill styling.
- **`ExerciseMap.jsx`**: Leaflet + CartoDB Positron tiles, bbox 108–127°E / 8–30°N. Performer palette is PRC red / ROC green / US blue / JP teal / MULTI grey. Popups carry per-row admin actions (Edit / Dismiss) gated by `READ_ONLY`. Selecting a list row pans + opens that marker's popup via the internal `PanController` helper. Uses react-leaflet v4 with `--legacy-peer-deps` install — see [[react-leaflet-v19-gotcha]]; `StrictMode` is intentionally off in `index.js` for Leaflet compatibility, do not re-enable.
- **`ExerciseReviewQueue.jsx`**: Modal-style analyst flow for pending candidates. Exports the editable-fields shape (`exerciseDraftFrom`, `isExerciseDraftDirty`, `buildExercisePatch`, `ExerciseFieldsGrid`) so the same UI is reused for approved-row editing — keep these in lockstep with `ExercisePatch` in `api/routes/military.py`. PATCH body must contain only changed fields: sending `name_en=''` on a name_zh-only row would NULL `canonical_name` server-side and break future auto-merge. On approve, the server auto-merges other pending rows sharing the same `canonical_name`.
- **`ExerciseEditModal.jsx`**: Save + Dismiss modal for already-approved exercises. Reuses the same field grid as the candidate review card; approve / merge actions deliberately omitted (those are pending-only). Triggered by the ✎ icon on map popups and list rows; both gated by `!READ_ONLY`.
- **`PollsTab.jsx`**: top-level tab (Phase 2d, 2026-05-26). Layout: three flagship trend strips pinned at top (`FLAGSHIPS` array — NCCU identity, NCCU unification, Lai approval cross-pollster), a two-tier "Other Trackers" selector (`buildOtherFamilies` + `OtherTrackers`) — tier-1 family **dropdown**, tier-2 question pills within the chosen family (both hide flagship keys + zero-wave questions so nothing advertises a dead-end click); picking a family auto-opens its most-used question, on-demand 4th chart strip when a question is selected, recent polls feed at the bottom. `buildOtherFamilies` orders families by the canonical `FAMILY_LABELS` sequence (stable, not by volatile counts) and is the single source for both the render and top-question resolution. Exports `pollsterChipColour(bias, slug)` — side-aware for `state_official` (slug='mac' → DPP green for TW exec; any other slug with state_official bias → red, future-proofing PRC state pollsters). Also exports `PollsterChip` and `FAMILY_LABELS` so PollReviewQueue / PollEntryModal stay visually consistent. `OPTION_PALETTES` is keyed by `question_key`; index order MUST match the seeded `option_order` in `nccu_esc_seed.json` because the chart pivots by position, not by label. `pivotByQuestion` anchors waves on `fielded_end || fielded_start` so annual series (NCCU) plot at the year-final publication date rather than Jan 1. **Colour resolution is two-mode**: if ANY option carries a party/`colour_override` (from the `/by-question` option fields), the chart enters *party mode* — party drives the line colour (`PARTY_COLOURS` from `partyColours.js`) and, on cross-pollster overlays, the **pollster becomes the dash** (so one candidate keeps one hue across pollsters); options without a party fall back to the positional palette. Otherwise *legacy mode* (single-pollster → positional `OPTION_PALETTES` keyed by `question_key` with index order matching `nccu_esc_seed.json`; cross-pollster → `pollsterChipColour` by bias, option → dash). `PollTrendChart` shows a small admin-only **Colours** button (opens `<PollColourModal />`) when `onOpenColours` is passed. `ProvenanceChip` distinguishes the three poll entry paths documented in [[database]] (AI / Backfill / Manual). Three admin buttons: header `+ Add poll` opens `<PollEntryModal />`, `✎ Review` opens `<PollReviewQueue />` (badged when pending > 0); per-chart `Colours` opens `<PollColourModal />` (all hidden under `READ_ONLY`).
- **`PollReviewQueue.jsx`**: modal approval flow for pending poll candidates. Per-question dropdown picks an existing `question_key` OR creates one inline (family + scale_type + canonical wording); per-question Skip toggle drops subgroup / single-statistic factoids the analyst doesn't want as canonical questions (server rejects all-skipped approve with 400 — use Dismiss instead). Pollster dropdown carries a `+ Create new pollster…` branch (`<NewPollsterForm />` posts `/api/polls/pollsters`) for AI extractions whose `pollster_hint` resolved to `unknown`. Envelope edits applied as overrides on the approve body, so the server-side canonical-merge on `(pollster_id, fielded_start)` reflects the analyst's corrections, not the raw AI extraction. Merge dropdown is populated from recent approved polls (100-row pool); analyst-edited envelope fields are PATCH'd through before the merge so the dismissed-into-merged row carries the correction for audit. `validateResolutions` and `buildApproveBody` are pure; the server-side validators in `api/routes/polls.py` are the source of truth — mirror enum changes there to the constants at the top of this file.
- **`PollEntryModal.jsx`**: manual-entry fallback for analyst-spotted polls that the AI pipeline missed (TVBS PDF the scraper hasn't indexed yet, off-roster ad-hoc pollster, etc.). Skips the pending state entirely — server lands the row as `approved` in one transaction. Reuses the same pollster picker + inline create branch and the same question_key picker + inline create branch as `PollReviewQueue`, but each question carries its own editable options grid (label_zh / label_en / percentage) rather than a read-only AI extraction. `validateDraft` mirrors the server's contract: pollster + fielded_start required, ≥1 question, ≥1 option per question, percentage numeric 0–100, label_zh OR label_en required per option, new-key fields complete when "create new" branch selected. Submits to `POST /api/polls/`; 409 surfaces inline if a twin already exists on `(pollster_id, fielded_start)` (analyst should PATCH the existing one instead).
- **`PollColourModal.jsx`**: per-chart analyst tool to assign candidate/party colours. Lists the chart's distinct option labels (from the `payload` it's handed); each row has an **Auto** option (resolve via key_figures / positional, shown with the auto-derived party in parens), the party dropdown (`PARTY_ORDER` / `PARTY_LABELS`), and a `#RRGGBB` override input (wins over party — for independents / clashes). Seeds initial state from `GET /api/polls/option-parties` (explicit assignments only, so Auto vs pinned is distinguishable). Save PUTs only changed rows; setting a row to Auto with no override deletes the assignment server-side. On save, `PollsTab` refetches just that question (`refreshQuestion`). Palette lives in `partyColours.js` (the single source of truth, also used by `KeyFigures.jsx` `figureAccent`).
- **`DiplomacyTab.jsx`**: top-level tab (Phase 2c). Third-country stance on the Taiwan question — a SEPARATE axis from cross-strait sentiment, encoded **red (pro-Beijing) ↔ green (pro-Taipei)** (reuses the locked side conventions; deliberately NOT the purple/amber sentiment axis). Layout: intro + KPI strip + stance-distribution bar/legend (`StanceBar`) + `<DiplomacyMap>` (left) and a stance-sorted country list (right) + a drill-in `SelectedDetail` panel. `SelectedDetail` fetches `/statements` with **`days=730`** (matching the map's `stale_days`) so its "Official statements (averaged)" list IS exactly the set averaged into the fill, plus the non-official "voices". Shared `StatementRow`/`StatementList` (no duplicated row markup). `◉/○ Voices pins` toggle drives the pin layer; admin-only `✎ Review` button (gold when pending) opens `<DiplomacyReviewQueue>`, badged from the cheap `/candidates/count`. `TIER_LABEL` is imported from `DiplomacyMap` (single source, like `BAND_*`). `OFFICIAL_TIERS = {government, head_of_state}`. Admin-only per-statement **✎** in `SelectedDetail` (gated by `!READ_ONLY`) opens `<DiplomacyEditModal>` for post-approval fixes; on save/dismiss it `reloadMap()`s (the fill may shift across a band) and refetches the drill-in via a `detailNonce`.
- **`DiplomacyMap.jsx`**: react-leaflet **choropleth** (world `<GeoJSON>` from `/geo/world-110m.geojson`, joined on `iso_a2`) — fill by official-posture band, **gold dashed border for `divergent`**, a separate non-interactive overlay layer for the selected country (so clicks don't remount the base layer). Microstates NE-110m omits as polygons (SG/VA/PW/MH/NR/TV/KI/KN/LC/VC) render as fill markers at hardcoded centroids. **Voices-pin layer** (`showPins`): a CircleMarker at each polygon country's NE label point (`lx`/`ly` from the geojson), coloured by `pins_label` (the non-official aggregate) — a green pin on a red fill = the divergence headline. Exports `BAND_COLOUR`/`BAND_LABEL`/`BAND_ORDER`/`TIER_LABEL` (shared by the tab + review queue). Basemap is built by `scripts/build_world_geojson.py` (trims NE 1:110m admin0 to `iso_a2`/`name`/`lx`/`ly`; re-run only to refresh). Same react-leaflet v4 + StrictMode-off caveats as `ExerciseMap` — see [[react-leaflet-v19-gotcha]].
- **`DiplomacyReviewQueue.jsx`**: modal candidate triage cribbed from `ExerciseReviewQueue`. **Collapsible per-country groups** (only expanded groups render cards — keeps the DOM light across the thousands of pending), sorted biggest-backlog-first. Per statement: original Chinese shown for translation review + editable grid (`DiplomacyFieldsGrid` — country/iso/source_side/speaker/tier/`stance` with live band-colour preview/stated_date/statement_en) + Approve (saves edits first if dirty) / Dismiss. No merge picker — approve-best + dismiss-dupes covers it (the endpoint exists for later). `buildDiplomacyPatch` sends only changed fields; mirror enum changes to `StatementPatch` in `api/routes/diplomacy.py`. Exports the editable-fields shape (`DiplomacyFieldsGrid`, `diplomacyDraftFrom`, `isDiplomacyDraftDirty`, `buildDiplomacyPatch`) so `DiplomacyEditModal` reuses the exact same edit UI on approved rows — the same lockstep `ExerciseReviewQueue` keeps with `ExerciseEditModal`.
- **`DiplomacyEditModal.jsx`**: Save + Dismiss modal for an already-approved diplomacy statement — the analyst path for fixing a rogue categorisation after approval (the review queue is pending-only). Reuses `DiplomacyReviewQueue`'s exported field grid + patch helpers; approve / merge deliberately omitted (those are pending-only). Triggered by the ✎ on each statement row in `DiplomacyTab`'s `SelectedDetail`, gated by `!READ_ONLY`. The `PATCH` / dismiss endpoints are status-agnostic, so no API change was needed. Mirrors `ExerciseEditModal`.
- **`FlashTraffic.jsx`**: priority signals section — renders full `ArticleCard` components, inverted colour scheme (`.signal-inverted` CSS class).
- **`SocialPulse.jsx`**: accepts `column` prop — in column mode (right-hand aside) always expanded, vertical stack layout; in default inline mode, collapsible with two-column Weibo/PTT panel. Weibo shows only cross-strait relevant items. Inline translation correction via pencil icon (hidden in read-only build). Override colour highlight is also hidden in read-only build.
- **`KeyFigures.jsx`**: horizontal scrollable row of cards above SocialPulse; each card shows portrait (images in `frontend/public/figures/`, initials fallback with party colour), name, role, latest approved statement; pencil icon (amber when candidates pending) opens per-card curation modal; hidden in read-only build.
- **`AboutModal.jsx`**: triggered from header (desktop) and mobile header "i" button; explains methodology, sentiment axis, source bias taxonomy, AI pipeline, author bio. Follows CSS variable conventions.
- **`SourceBadge.jsx`**: colour-coded by `bias` prop — `SOURCE_ABBREV` map covers all active sources; multi-feed publications collapse to a shared abbreviation (e.g. all CT sections → `CT`).
- **`FilterBar.jsx`**: topic / sentiment / source_place / bias / entity / escalation / search filters. Source place options: PRC / Taiwan / HK/Macao (`hk`) / International (`intl`).
- **`hooks/useWindowWidth.js`**: returns `window.innerWidth`, updates on resize. Used in `App.js` to derive `isMobile = windowWidth < 768`.
- **`ThemeToggle.jsx`** — light/dark theme switcher in the header. **`TopicPill.jsx`** — inline topic category label used in `ArticleCard`.

## Build env caveat

Never run `npm run build` (admin bundle) without sourcing `.env` — the admin bundle bakes in `REACT_APP_ADMIN_TOKEN` at build time. A plain `npm run build` ships an empty token; every write request then fails auth.

For iterating fast: `set -a && source .env && set +a && cd frontend && REACT_APP_ADMIN_TOKEN="$ADMIN_TOKEN" npm run build`. Otherwise use `bash server_deploy.sh` which handles env sourcing. The public build (`npm run build:public`) intentionally has no token and is safe to run without env.
